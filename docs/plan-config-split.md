# Plan: phonewave.yaml Config/State Split (Phase 1 Only)

## Problem

`phonewave.yaml` mixes git-trackable declarations with environment-specific state:
- Endpoint definitions (produces/consumes) — shareable, derived from SKILL.md
- Absolute repo paths — machine-specific (WriteConfig already relativizes, but old files exist)
- Derived routes — computable from endpoints at runtime
- last_synced timestamp — machine-specific

## Design Decision

**Phase 1 only**: Split the file into two, keep the existing multi-repo model and
LoadConfig/WriteConfig as facade. No behavioral changes to any command.

### phonewave.yaml (git-tracked)

Contains repositories and their endpoint declarations. Paths are relative (already
implemented in WriteConfig). Routes and last_synced are removed.

```yaml
# phonewave.yaml — auto-generated by phonewave
repositories:
    - path: .
      endpoints:
        - dir: .expedition
          produces: [report, feedback]
          consumes: [specification, feedback]
        - dir: .gate
          produces: [feedback, convergence]
          consumes: [report]
        - dir: .siren
          produces: [specification, report, feedback]
          consumes: [feedback, convergence]
```

### .phonewave/.run/resolved.yaml (gitignored)

Contains derived routes and local metadata. Regenerated by `sync` and `init`.

```yaml
last_synced: 2026-03-08T10:00:00Z
routes:
    - kind: report
      from: .expedition/outbox
      to: [.gate/inbox]
      scope: same_repository
      repo_path: .
    ...
```

## Changes

### domain layer
1. Add `ResolvedStateFile = "resolved.yaml"` constant
2. New `ResolvedState` struct: `LastSynced time.Time` + `Routes []RouteConfig`

### session layer
3. `WriteConfig(path, cfg)` → now writes two files:
   - phonewave.yaml: repositories only (no routes, no last_synced)
   - .phonewave/.run/resolved.yaml: routes + last_synced
4. `LoadConfig(path)` → reads both files, merges into existing Config struct
   - If resolved.yaml missing: derive routes from endpoints (graceful degradation)
5. Backward compat: if old phonewave.yaml has `routes:` key, migrate on next write

### cmd layer
No changes. All commands continue to use LoadConfig/WriteConfig facade.

## Files to modify
- internal/domain/phonewave.go (add const + type)
- internal/session/config.go (modify LoadConfig, WriteConfig)
- internal/session/config_test.go (update tests)

## Risk Assessment
- Low: facade preserves all existing command behavior
- Migration: old format auto-detected and split on next write
- Graceful: missing resolved.yaml → routes derived at runtime from endpoints
