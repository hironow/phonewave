# OTel Backend Configuration

phonewave supports OpenTelemetry trace export to Jaeger (local) and Weave (Weights & Biases).

## Jaeger (Local Development)

```bash
# Start Jaeger
docker run -d --name jaeger \
  -p 16686:16686 \
  -p 4318:4318 \
  jaegertracing/jaeger:latest

# Initialize with Jaeger backend
phonewave init --otel-backend=jaeger ./repo-a ./repo-b

# Verify traces at http://localhost:16686
```

## Weave (Weights & Biases)

```bash
# Set API key
export WANDB_API_KEY=your-key-here

# Initialize with Weave backend
phonewave init --otel-backend=weave \
  --otel-entity=my-team \
  --otel-project=my-project \
  ./repo-a ./repo-b

# Verify traces at https://wandb.ai/<entity>/<project>/weave
```

## Manual Override

Environment variables always take precedence over `.otel.env`:

```bash
OTEL_EXPORTER_OTLP_ENDPOINT=http://custom:4318 phonewave run
```

## .otel.env File

Generated by `phonewave init --otel-backend` at `.phonewave/.otel.env`.
Format: `KEY=VALUE` with `${VAR}` expansion and `#` comments.

## Weave Verification

1. `export WANDB_API_KEY=your-key-here`
2. `phonewave init --otel-backend=weave --otel-entity=my-team --otel-project=my-project ./repo`
3. `phonewave run`
4. Open `https://wandb.ai/<entity>/<project>/weave` and verify spans appear

## GenAI Semantic Conventions

phonewave does not invoke LLMs directly and does not emit `gen_ai.*` span attributes.
Tools that invoke Claude (sightjack, paintress) emit:

- `gen_ai.operation.name=chat`
- `gen_ai.system=anthropic`
- `gen_ai.request.model=<model>`
